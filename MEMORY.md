**【CTO 最高指令：赛博观测站日常发文 SOP】**

**主编请注意，我们的网站架构已经彻底定型。为了避免触发云端 CI/CD 的毁灭性报错，在你未来接收到任何“发布新文章”的指令时，必须且只能严格按照以下 3 步协议执行，禁止任何形式的自我发挥！**

**第一步：标准化的 MDX 创作（内容层）**

- 在 /home/admin/my-ai-blog/docs 目录下创建新的 .mdx 文件。

- **【绝对红线】**：文件开头必须包含标准的 Frontmatter，用来定义网页标题和侧边栏显示的 Emoji 标题。正文第一行禁止使用 # 标题（防止重叠）。

- **模板规范**：

  ```
  ---
  title: "这里是文章的完整大标题"
  sidebarTitle: "🚀 这里是带 Emoji 的侧边栏短标题"
  ---
  这里开始写正文内容...
  ```

**第二步：极简的 JSON 路由注入（配置层）**

- 读取并修改 docs.json。
- 定位到 "navigation": { "tabs": [ ... ] } 结构。
- 找到目标频道的 pages 数组（如 "tab": "🦀 龙虾进化论" 下的 pages）。
- **【生死红线】**：在 pages 数组中添加新页面的路径时，**只能填入纯字符串**（例如 "new-article-name"，不要 .mdx 后缀）。**绝对禁止**在 pages 数组里使用 { "page": "...", "title": "..." } 的对象格式！否则网站标题栏会全部崩溃！

**第三步：标准化部署（发布层）**
注意: 必须推送到main分支里

- 依次执行：git add . -> git commit -m "docs: 发布新文章[文章名]" -> git push origin main。
